# 音乐后台管理系统项目大纲

## 项目概述
基于Java Web后端开发技术课程要求，构建一个功能完整的音乐流媒体后台管理系统，采用前后端分离架构。

## 技术架构

### 前端技术栈
- **HTML5 + CSS3**：页面结构和样式
- **JavaScript ES6+**：交互逻辑
- **Tailwind CSS**：样式框架
- **动画库**：Anime.js, Pixi.js, Matter.js
- **图表库**：ECharts.js
- **字体**：Noto Sans SC, JetBrains Mono

### 后端技术栈（模拟）
- **Node.js + Express**：轻量级后端服务
- **JSON文件**：数据存储（模拟数据库）
- **RESTful API**：前后端通信
- **JWT**：身份认证

## 页面功能详细规划

### 1. 登录页面 (index.html)
- **功能**：管理员身份验证
- **内容**：
  - 系统logo和标题
  - 登录表单（用户名、密码）
  - 记住密码选项
  - 登录按钮和错误提示
- **交互**：
  - 表单验证
  - 登录动画效果
  - 错误状态显示

### 2. 主仪表盘 (dashboard.html)
- **功能**：系统概览和数据可视化
- **内容**：
  - 统计卡片（用户总数、歌曲数量、播放量等）
  - 实时数据图表
  - 系统状态监控
  - 快速操作入口
- **交互**：
  - 动态数据更新
  - 图表交互
  - 卡片悬停效果

### 3. 音乐管理页面 (music-management.html)
- **功能**：音乐内容管理
- **内容**：
  - 歌手管理（列表、添加、编辑、删除）
  - 专辑管理（列表、添加、编辑、删除）
  - 歌曲管理（列表、上传、编辑、删除）
  - 搜索和筛选功能
- **交互**：
  - 文件上传功能
  - 批量操作
  - 模态框编辑

### 4. 用户管理页面 (user-management.html)
- **功能**：用户账户管理
- **内容**：
  - 用户列表（头像、信息、状态）
  - 用户详情查看
  - 用户行为分析
  - 批量用户操作
- **交互**：
  - 用户状态切换
  - 数据筛选排序
  - 分页功能

### 5. 歌单管理页面 (playlist-management.html)
- **功能**：歌单内容管理
- **内容**：
  - 歌单列表展示
  - 歌单详情管理
  - 歌曲添加/删除
  - 推荐设置
- **交互**：
  - 拖拽排序
  - 歌曲搜索添加
  - 批量操作

### 6. 系统日志页面 (system-logs.html)
- **功能**：操作日志和审计
- **内容**：
  - 日志列表（时间、用户、操作、结果）
  - 日志搜索筛选
  - 日志导出功能
  - 异常监控
- **交互**：
  - 日志详情查看
  - 高级搜索
  - 数据导出

## 数据模型设计

### 用户数据模型
```json
{
  "id": "用户ID",
  "username": "用户名",
  "email": "邮箱",
  "avatar": "头像URL",
  "status": "状态（active/inactive）",
  "role": "角色（admin/user）",
  "createdAt": "创建时间",
  "lastLogin": "最后登录时间"
}
```

### 音乐数据模型
```json
{
  "id": "歌曲ID",
  "title": "歌曲标题",
  "artist": "歌手",
  "album": "专辑",
  "duration": "时长",
  "genre": "类型",
  "fileUrl": "文件URL",
  "coverUrl": "封面URL",
  "playCount": "播放次数",
  "uploadDate": "上传日期"
}
```

### 歌单数据模型
```json
{
  "id": "歌单ID",
  "name": "歌单名称",
  "description": "描述",
  "creator": "创建者",
  "coverUrl": "封面URL",
  "songs": ["歌曲ID数组"],
  "isPublic": "是否公开",
  "createDate": "创建日期",
  "playCount": "播放次数"
}
```
